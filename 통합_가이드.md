# ShoppingApp 저장소에 백엔드 추가하는 방법

이 zip 안의 내용을 기존 `ShoppingApp` 폴더(로컬 디스크 C: > 앱 개발 > ShoppingApp_deeplink2 > ShoppingApp)에
그대로 덮어씌우면(병합하면) 됩니다.

## 적용 후 최종 구조

```
ShoppingApp/                       ← 기존 그대로
├── app/                           ← 기존 안드로이드 앱 (그대로 유지, 손대지 않음)
├── gradle/
├── build.gradle
├── settings.gradle
├── .gitignore
├── gradle.properties
├── local.properties
│
├── backend/                       ← ★ 새로 추가되는 폴더
│   ├── shared/
│   │   ├── normalize.js
│   │   ├── deeplink.js
│   │   └── priceHistory.js
│   ├── collector/
│   │   └── collect.js
│   ├── search-function/
│   │   ├── src/index.js
│   │   └── wrangler.toml
│   └── package.json
│
└── .github/
    └── workflows/
        ├── (기존에 있던 안드로이드 빌드 워크플로우 — 그대로 유지)
        └── collect-prices.yml    ← ★ 새로 추가되는 파일
```

## 적용 방법 (단계별)

1. 이 zip을 압축 해제합니다.
2. 압축 해제한 폴더 안의 `backend` 폴더를 통째로 복사해서,
   기존 `ShoppingApp` 폴더 바로 안에 붙여넣습니다.
   (기존 `app`, `gradle` 폴더와 같은 위치, 같은 깊이입니다.)
3. 압축 해제한 폴더 안의 `.github/workflows/collect-prices.yml` 파일을
   기존 `ShoppingApp/.github/workflows/` 폴더 안에 복사해 넣습니다.
   - 이미 그 폴더에 다른 워크플로우 파일(예: 안드로이드 빌드용)이 있어도
     **파일명이 다르므로 충돌 없이 같이 존재**할 수 있습니다.
4. 그대로 git add, commit, push 하면 됩니다.

```bash
cd "C:\앱 개발\ShoppingApp_deeplink2\ShoppingApp"
git add backend/ .github/workflows/collect-prices.yml
git commit -m "가격 비교 백엔드(수집/검색) 추가"
git push
```

## 그 다음 — API 키 등록

GitHub 저장소 페이지에서:
`Settings` → `Secrets and variables` → `Actions` → `New repository secret`

다음 4개를 등록합니다 (발급받는 즉시):
- `COUPANG_ACCESS_KEY`
- `COUPANG_SECRET_KEY`
- `NAVER_CLIENT_ID`
- `NAVER_CLIENT_SECRET`

등록하고 나면 코드를 전혀 손대지 않아도, 다음 스케줄(3시간마다) 또는
`Actions` 탭에서 `Run workflow` 버튼을 눌러 즉시 실제 데이터 수집이 시작됩니다.
